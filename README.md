# 🍞 CookingOverhaul — реалистичная кулинария в Minecraft

**CookingOverhaul** — это плагин для серверов Minecraft (Paper/Spigot 1.16+), который полностью заменяет стандартный способ получения хлеба. Забудьте о крафте «3 пшеницы = 1 хлеб» — теперь вы пройдёте все этапы: помол зерна в муку, замешивание теста и выпечку в печи или коптильне.

> ⚠️ **Плагин находится в активной разработке.** В будущем появятся новые рецепты, кухонная утварь и расширенная система кулинарии.

## 🚀 Возможности

- **Реалистичная переработка пшеницы**  
  Получение муки с помощью ножниц (имитация помола).  
- **Замешивание теста**  
  Крафт в верстаке из муки и ведра воды.  
- **Выпечка хлеба**  
  Два способа: обычная печь (300 тиков) или коптильня (150 тиков) — быстрее, но требует больше ресурсов на изготовление.  
- **Полное отключение ванильного рецепта хлеба**  
  Игроки не смогут обойти систему старым способом — только новый реалистичный путь.  
- **Защита от случайного использования кастомных предметов**  
  Муку (костную муку) нельзя использовать для удобрения, а тесто (белый краситель) — для окрашивания.  
- **Автоматическое восстановление рецептов**  
  При выгрузке плагина ванильный рецепт хлеба возвращается.

## 📦 Установка

1. Скачайте последний релиз плагина (`CookingOverhaul.jar`).
2. Поместите файл в папку `plugins` вашего сервера.
3. Перезапустите сервер (или используйте `/reload`, но с осторожностью).
4. Готово! Ванильный рецепт хлеба будет удалён, и все новые рецепты добавлены автоматически.

> **Требования:**  
> - Сервер Paper / Spigot 1.16.5 и выше (используется `PersistentDataContainer`).  
> - Java 8 или выше.

## 🌾 Игровой процесс

### 🍚 Получение муки
- Возьмите **пшеницу** в основную руку.
- Возьмите **ножницы** во вторую руку.
- Нажмите **ПКМ (правую кнопку мыши)** по воздуху.
- Вся стопка пшеницы превратится в эквивалентное количество муки, ножницы потеряют 1 единицу прочности (возможна поломка), а вы услышите звук стрижки.

> **Мука** — это кастомный предмет (костная мука с именем `Мука`), который **нельзя** использовать для удобрения земли.

### 🥖 Замешивание теста
Откройте верстак и разместите предметы **в любом порядке** (бесформенный рецепт):

| Ингредиенты | → Результат |
|------------|------------|
| 3 муки + 1 ведро воды | 1 тесто |

После крафта ведро заменится на пустое ведро.

> **Тесто** — кастомный предмет (белый краситель с именем `Тесто`), который **нельзя** применить как краситель.

### 🔥 Выпечка хлеба
Готовое тесто поместите в печь или коптильню:

- **Печь** → 300 тиков (15 секунд) → 1 хлеб
- **Коптильня** → 150 тиков (7.5 секунд) → 1 хлеб

На выходе вы получите обычный ванильный хлеб, пригодный для еды и других крафтов.

## 🔐 Разрешения и команды

На данный момент плагин не добавляет команд и не использует систему прав. Все механики работают автоматически для всех игроков.

## 🧩 Технические детали

- Все кастомные предметы (мука, тесто) хранят метку в `PersistentDataContainer` с ключом `NamespacedKey`, что гарантирует корректную идентификацию даже после перезагрузки сервера.
- Рецепты регистрируются с помощью `RecipeChoice.ExactChoice` (точное совпадение, включая метаданные) для предотвращения подмены предметов.
- Прослушиваются события:
  - `PlayerInteractEvent` – создание муки и блокировка использования теста/муки.
  - `BlockFertilizeEvent` – предотвращение удобрения кастомной мукой.
- При выключении плагина оригинальный рецепт хлеба восстанавливается, чтобы не нарушать работу других систем.

## 📖 Примеры использования

1. **Добыча муки**  
   Вырастите пшеницу, возьмите ножницы и комбайном «перемолите» её в муку прямо в инвентаре.

2. **Массовое производство**  
   Поставьте автоматическую ферму пшеницы, подавайте её в верстак вместе с ведром воды — получайте тесто, которое затем отправляется в автопечь или коптильню.

3. **Использование в кулинарных модах**  
   Мука и тесто могут быть использованы другими плагинами, которые поддерживают кастомные предметы (через `NamespacedKey`).

## 🛠️ Для разработчиков

Исходный код написан на Java с использованием Bukkit API.  
Ключевые особенности реализации:
- Класс `CookingOverhaul` — главный класс, регистрация рецептов и слушателей.
- Методы `getFlour()` и `getDough()` создают кастомные предметы с уникальным ключом `NamespacedKey`.
- Удаление ванильного рецепта `removeVanillaBreadRecipe()` и восстановление `restoreVanillaBreadRecipe()` на `onDisable()`.
- Проверка кастомности через `isCustomItem()`.

Плагин не имеет конфигурации, все параметры (длительность обжига, количество ингредиентов) заданы в коде.
